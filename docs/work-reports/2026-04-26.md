## Task 2: Diagnose + Fix Dead GHL Webhook URL
**Project:** Brio Website — GHL Lead Pipeline
**Date:** 2026-04-26
**Hours:** 1.0
**Category:** Bug Fix

**Summary:**
User reported live form submissions were not appearing in GHL. Direct curl to the hardcoded webhook URL returned `HTTP 200 {"status":"Success: test request received"}` — appearing healthy — yet no contact was created. ## Task 3: Add Budget / Start-Date / Financing Fields to Shared LeadForm
**Project:** Brio Website — LeadForm Component
**Date:** 2026-04-26
**Hours:** 0.75
**Category:** Feature

**Summary:**
The website LeadForm only collected name/phone/email/address/description, leaving four GHL custom fields (`estimated_budget`, `project_start_date`, `financing_interest`, `sms_opt_in`) empty on every contact. Workflow #9 (urgency/financing tagger) had no data to branch on. Added two optional dropdowns (Budget and Start Date) pre-selected to "Not sure yet" so they require zero clicks for users who don't know, plus a financing-interest checkbox. SMS consent is now implicit via a disclaimer line under the submit button (one less action), with `sms_opt_in: "Yes"` always sent. Both `stacked` and `compact` variants were updated so all 14+ service pages and the city pages get the new fields automatically.

**Key Changes:**
- Added `budgetOptions` prop with sensible 6-bucket default ($25K → $250K+) and `BudgetOption` type export
- Added Budget select, Start Date select, and Financing checkbox with appropriate `defaultValue` and labels in both variants
- Added implicit-consent disclaimer paragraph under submit button: *"By clicking, you agree to receive text messages and emails from BRIO Construction. Reply STOP to opt out anytime."*
- Extended `LeadPayload` interface and `submitLead()` body to pass through the four new fields, with `sms_opt_in` defaulting to `"Yes"` and `financing_interest` mapping boolean → `"Yes"`/`"No"`

**Commits:** `dc8f43f`
