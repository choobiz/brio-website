## Task 1: Polish Project Detail + Index Pages
**Project:** Brio Website — Projects
**Date:** 2026-04-17
**Hours:** 1.0
**Category:** Enhancement

**Summary:**
Reviewed and polished the 7 project detail pages plus the projects index. Fixed five distinct issues: generic "Commercial"/"Residential" labels, an image-reuse bug on 3-section projects, an arbitrary showcase image that appeared on every project, concatenated index excerpts, and always-the-same related projects. Shipped behind one commit and deployed to production.

**Key Changes:**
- Added `projectType` field per project (`Commercial Tenant Improvement`, `New Duplex Construction`, `Condo Renovation`, `Home Renovation`) so the detail page shows the specific project type instead of the filter-level `type` field
- Added `showcaseImage?: boolean` flag — only coffeeshop now renders the full-width showcase image; other projects no longer show an arbitrary `img[1]` slot as a break
- Fixed "Built with Intention" image duplication: it used to pick `img[img.length - 1]`, which on 3-section projects (Trinity, East Broadway, LoLo, Jefferson) was the same image already shown in the final section. Now uses `heroImage` which is always distinct
- Cleaned up projects index excerpts from a concatenated blob of `name + subtitle + "Project Type:" + "Project Year:" + description-prefix` to just a clean subtitle, plus a small `projectType · year` meta line under each card title
- Related-projects logic now prefers same-type projects first (commercial → commercial, residential → residential) before falling back to others — still capped at 3
