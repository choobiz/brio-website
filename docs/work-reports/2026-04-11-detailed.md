## Task 1: Rebuild Header Dropdown Navigation

### Problem
The Navbar had flat navigation links with no dropdown menus. The live WordPress site has dropdown menus under "Our Services" (12 items) and the site also has city pages that need a dropdown under "Areas We Serve".

### Solution

#### File: `components/shared/Navbar.tsx`

**Change 1: Complete rewrite with dropdown system**
- Replaced simple `NAV_ITEMS` array with typed `NavItem[]` that supports optional `dropdown` property
- Navigation structure: Our Services (14 items from SERVICES constant), Projects, Areas We Serve (9 items from AREAS constant), About, Blog, Financing
- Desktop: hover-to-open dropdowns with `DesktopDropdown` sub-component using `onMouseEnter`/`onMouseLeave` with 150ms timeout to prevent flicker
- Mobile: accordion-style expandable sections using `mobileExpanded` state, with ChevronDown rotation animation
- Added "Get a Quote" CTA button linking to `/about-us#get-in-touch`
- Imports SERVICES and AREAS from `@/lib/constants` for dropdown items

**Key implementation details:**
- `DesktopDropdown` uses a `useRef` timeout to debounce mouse leave events (150ms delay)
- Dropdown positioned with `absolute top-full left-0 pt-2 z-50` and `min-w-[260px]`
- Mobile menu closes on any link click via `onClick={() => setMobileOpen(false)}`
- Height increased from 64px to 72px for better visual balance

### Verification
- `npx next build` — clean build, no errors
- Dev server started on port 3001
