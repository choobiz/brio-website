## Task 1: Polish Project Detail + Index Pages

### Problem

Five separate issues on the `/projects` index and `/projects/[slug]` detail pages:

1. **Generic project type label.** Detail page title block showed `Project Type: Commercial` or `Project Type: Residential`. The live WordPress site and the projects index excerpts used specific labels (`Commercial Tenant Improvement`, `New Duplex Construction`, `Condo Renovation`, `Home Renovation`). The `Project` interface only had a filter-level `type: "Commercial" | "Residential"` field.

2. **Full-width showcase image rendered on every project.** The detail template had a "PAUL LE CAFÉ counter" section gated only on `img[1]` existence. Since every project has at least 5 images, the condition was always true — so every project showed an arbitrary `img[1]` as a full-width hero-style break, not just coffeeshop.

3. **"Built with Intention" duplicated the previous section's image.** The final CTA section used `img[img.length - 1] || heroImage`. On 3-section projects (Trinity/East Broadway/LoLo/Jefferson, all 5 images), `s[2]` used `img[4]` and `img.length - 1 = 4` — so `img[4]` was shown twice back-to-back.

4. **Projects index excerpts were a concatenated mess.** Example: `"Downtown Coffeeshop Renovation Bringing a refined European café experience to the heart of Downtown Vancouver Project Type: Commercial Tenant Improvement Project Year: 2024-25 Bringing a..."` — name, subtitle, type, year, and description prefix all jammed together, clamped to 4 lines.

5. **Related projects always showed the first 3 non-current projects.** The filter was `PROJECTS.filter(p => p.slug !== slug).slice(0, 3)` — no type preference, no variation.

### Root Cause

The `Project` interface lacked (a) a display-level project-type field and (b) an opt-in flag for the showcase section. The detail template was built for the coffeeshop content shape (7 images, 5 sections) and didn't degrade cleanly for 3-section projects. The index page had its own hardcoded `PROJECTS` array with excerpt text that was clearly scraped/concatenated rather than authored.

### Solution

#### File: `lib/projects.ts`

**Change 1: Extended `Project` interface**
- Location: interface declaration at top of file
- Added `projectType: string` (required) and `showcaseImage?: boolean` (optional)

```ts
// Before
export interface Project {
  slug: string;
  name: string;
  subtitle: string;
  type: "Commercial" | "Residential";
  year: string;
  description: string;
  sections: ProjectSection[];
  ctaText: string;
  ctaLink: string;
  heroImage?: string;
  images?: string[];
  youtubeId?: string;
}

// After
export interface Project {
  slug: string;
  name: string;
  subtitle: string;
  type: "Commercial" | "Residential";
  projectType: string;
  year: string;
  description: string;
  sections: ProjectSection[];
  ctaText: string;
  ctaLink: string;
  heroImage?: string;
  images?: string[];
  youtubeId?: string;
  showcaseImage?: boolean;
}
```

#### File: `app/projects/[slug]/page.tsx`

**Change 1: Render `projectType` instead of `type`**
- Location: title-below-hero section
```tsx
// Before
<p className="text-text-muted text-[13px]">Project Type: {project.type}</p>

// After
<p className="text-text-muted text-[13px]">Project Type: {project.projectType}</p>
```

**Change 2: Gate showcase section behind `showcaseImage` flag**
- Location: "Full-width showcase image" section
```tsx
// Before
{img[1] && (
  <section className="relative w-full h-[300px] md:h-[450px]">
    <Image src={img[1]} alt={`${project.name} counter view`} ... />
  </section>
)}

// After
{project.showcaseImage && img[1] && (
  <section className="relative w-full h-[300px] md:h-[450px]">
    <Image src={img[1]} alt={`${project.name} showcase view`} ... />
  </section>
)}
```

**Change 3: Use `heroImage` for "Built with Intention" image**
- Location: closing CTA section
```tsx
// Before
<Image src={img[img.length - 1] || heroImage} alt="BRIO project showcase" ... />

// After
<Image src={heroImage} alt="BRIO project showcase" ... />
```
- Why: `heroImage` is a separate asset (`/images/projects/{slug}/hero.jpg`) that is never used in any section two-column block. Always distinct from `img[0..n]`.

**Change 4: Same-type-first related projects**
- Location: top of `ProjectDetailPage` render body
```tsx
// Before
const related = PROJECTS.filter((p) => p.slug !== slug).slice(0, 3);

// After
const others = PROJECTS.filter((p) => p.slug !== slug);
const related = [
  ...others.filter((p) => p.type === project.type),
  ...others.filter((p) => p.type !== project.type),
].slice(0, 3);
```

#### File: `app/projects/page.tsx`

**Change 1: Rewrote all 7 excerpts + added `projectType` to each card**
- Location: hardcoded `PROJECTS` array
- Each entry's `excerpt` is now the subtitle sentence only (one line). Added `projectType` field to each entry.

**Change 2: Card markup now renders projectType · year meta**
```tsx
// Before
<h3 className="... mb-2">{p.name}</h3>
<p className="text-text-body text-[13px] leading-relaxed mb-3 line-clamp-4">
  {p.excerpt}
</p>

// After
<h3 className="... mb-1">{p.name}</h3>
<p className="text-text-muted text-[11px] uppercase tracking-wide mb-2">
  {p.projectType} · {p.year}
</p>
<p className="text-text-body text-[13px] leading-relaxed mb-3">
  {p.excerpt}
</p>
```
- Dropped `line-clamp-4` since excerpts are now short single sentences.
- Reduced title `mb-2 → mb-1` to tighten spacing to the new meta line.
