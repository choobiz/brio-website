## P0 ÔÇö must-fix before DNS switch

### 1. `/about-us` has no `<h1>`
- `app/about-us/page.tsx` starts with a hero image (no text overlay), then jumps to `<h2>Our Core Values</h2>`.
- Four `<h2>`s on the page, zero `<h1>`. Breaks document outline and SEO.
- **Fix:** add an `<h1>` inside the hero section (e.g. "About BRIO Construction" or a brand-forward line like "Built with Confidence, Built by BRIO").

### 2. Houzz footer link returns HTTP 404
- `lib/constants.ts` Ôćĺ `COMPANY.social.houzz = "https://www.houzz.com/pro/builtbybrio"` ÔÇö 404.
- No canonical Houzz profile found for BRIO Construction (only individual photo pages with "by BRIO Construction" credit).
- **Fix:** either (a) remove the Houzz icon from the footer, (b) get a valid URL from the business owner, or (c) link to the Houzz photo search for "BRIO Construction" as a temporary pointer.

### 3. No `sitemap.xml`
- Required for search engines to efficiently discover all 64 routes.
- **Fix:** add `app/sitemap.ts` (Next.js auto-generates `/sitemap.xml`). Enumerate routes from `SERVICES`, `AREAS`, blog folder, and `PROJECTS` slugs.

### 4. No `robots.txt`
- Without it, search engines assume allow-all but there's no pointer to the sitemap.
- **Fix:** add `app/robots.ts` with `User-agent: *  Allow: /` and `Sitemap: https://builtbybrio.com/sitemap.xml`.

### 5. No canonical tags
- Every page lacks `<link rel="canonical">`. Important once DNS flips ÔÇö without canonicals, Google may index the Vercel preview domain alongside `builtbybrio.com`.
- **Fix:** set `metadataBase: new URL("https://builtbybrio.com")` in root `layout.tsx` metadata, then `alternates: { canonical: "/" }` on each page (or let Next.js infer).
